Раздел 3: Архитектура и жизненный цикл

Понимаем, как связаны классы движка и в каком порядке выполняется логика.

Главная Назад Дальше

Связи сущностей

Engine
  └─ sceneManager (alias: scenes)
      └─ active Scene
          └─ gameObjects: GameCollection
              └─ GameObject
                  └─ components: GameCollection
                      └─ BaseComponent descendants

Базовый контракт

Все сущности наследуют lifecycle-методы: init, start, update(dt), render(ctx), destroy.

Порядок кадра

  1. Engine.update(dt) обновляет активную сцену через SceneManager.
  2. Активная сцена обновляет объекты, объекты обновляют компоненты.
  3. CollisionSystem.updateCollisions() пересчитывает пересечения.
  4. Engine.renderFrame() рисует кадр с учетом камеры и масштабирования, затем loading overlay.
  5. InputSystem.update() очищает pressed/released (после логического шага).
Апдейты идут с фиксированным шагом, а отрисовка синхронизирована через requestAnimationFrame.

Lifecycle-таблица

Метод Когда Что обычно делать
init() После добавления в коллекцию Подготовка ссылок, регистрация систем, стартовые параметры
start() При первом запуске game loop Инициализация, зависящая от уже созданного мира
update(dt) Каждый логический шаг Геймплей, ввод, движение, правила
render(ctx) Каждый кадр Рисование (обычно через SpriteComponent)
destroy() При удалении Очистка подписок и удаление из систем
