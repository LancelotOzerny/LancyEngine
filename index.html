1) Быстрый старт

Минимальный путь, чтобы увидеть движок в работе.

  1. Установите зависимости: npm install
  2. Соберите движок: npm run build
  3. Запустите dev-режим с автоперезагрузкой: npm run develop
  4. Откройте index.html в корне проекта (BrowserSync поднимет локальный сервер)

Подключение бандлов

<link rel="stylesheet" href="./engine/styles.min.css">

<script src="./engine/math.min.js"></script>
<script src="./engine/common.min.js"></script>
<script src="./engine/core.min.js"></script>
<script src="./engine/components.min.js"></script>
<script src="./engine/ui.min.js"></script>
Важно: сначала core.min.js, потом components.min.js. Компоненты наследуются от BaseComponent, который объявлен в core.

2) Как устроен движок

Базовые сущности

  • Engine - singleton, хранит canvas, сцены, ввод, ассеты, камеру и игровой цикл.
  • Scene - контейнер игровых объектов (gameObjects).
  • GameObject - сущность в мире, содержит компоненты.
  • BaseComponent - поведение, которое можно подключить к объекту.

Стандартные компоненты

  • TransformComponent - позиция/поворот.
  • SpriteComponent - рендер спрайта или цветного прямоугольника.
  • ColliderComponent - AABB-коллизии, слои и маски.
  • PhysicComponent - скорость, силы, гравитация, резолв столкновений.
  • CameraComponent - следование за целью, dead-zone, bounds, shake.

Поток данных в кадре

InputSystem -> Scene.update(dt) -> GameObject.update(dt) -> Component.update(dt)
                                      -> CollisionSystem.updateCollisions()
                                      -> Engine.renderFrame() -> Scene.render(ctx)

3) Жизненный цикл объектов

Метод Когда вызывается Для чего использовать
init() После добавления и инициализации коллекции Привязка ресурсов, начальная настройка
start() Один раз при старте игрового цикла Стартовое состояние, кеш ссылок, камера
update(dt) Логический кадр (фиксированный шаг ~1/60 сек) Игровая логика, движение, обработка ввода
render(ctx) Каждый рендер-кадр Отрисовка (обычно делает SpriteComponent)
destroy() При удалении Очистка ссылок, отписки, удаление из коллекций

4) Основной API

Engine.instance

Метод/поле Назначение
init(selector, params) Инициализация canvas и параметров экрана.
run(prepareFunc, data) Подготовка данных + запуск цикла.
scenes.append(scene) Добавляет сцену в движок.
input.isKeyDown(code) Проверка удержания клавиши (например, KeyA).
screenToWorld(x, y) / worldToScreen(x, y) Преобразование координат мыши/мира.

Параметры Engine.init

Engine.instance.init("#game", {
  width: 1920,              // "дизайнерское" разрешение
  height: 1080,
  backgroundColor: "#0c1118",
  screenMode: "expand",     // "expand" | "contain"
  fitToWindow: true,
  autoResize: true,
  centerView: true,
  pixelArt: false,
  maxDeltaTime: 100
});

5) Первая игра (пошагово)

Ниже минимальный шаблон, который уже можно развивать в платформер/аркаду.

Шаг 1. Разметка

<canvas id="game"></canvas>

Шаг 2. Игровой объект игрока

class Player extends GameObject {
  constructor() {
    super();
    this.speed = 420;

    this.bindComponent(new SpriteComponent({
      width: 90,
      height: 90,
      color: "#2cb67d"
    }));

    this.bindComponent(new ColliderComponent({
      width: 90,
      height: 90,
      layer: 1
    }));
  }

  update(dt) {
    super.update(dt);
    const input = Engine.instance.input;
    let dx = 0;
    let dy = 0;

    if (input.isKeyDown("KeyA")) dx -= this.speed * dt;
    if (input.isKeyDown("KeyD")) dx += this.speed * dt;
    if (input.isKeyDown("KeyW")) dy -= this.speed * dt;
    if (input.isKeyDown("KeyS")) dy += this.speed * dt;

    this.transform.translate(dx, dy);
  }
}

Шаг 3. Сцена

class MainScene extends Scene {
  init() {
    this.player = new Player();
    this.player.transform.setPosition(300, 250);
    this.gameObjects.append(this.player);
  }
}

Шаг 4. Запуск

const engine = Engine.instance;

engine.init("#game", {
  width: 1920,
  height: 1080,
  backgroundColor: "#101820",
  screenMode: "expand"
});

engine.scenes.append(new MainScene());
engine.run(() => {
  console.log("Игра запущена");
}, { images: [] });
Если сначала нужен прототип механик, используйте цветные прямоугольники в SpriteComponent, а ассеты подключите позже.

6) Физика и коллизии

ColliderComponent

  • layer - слой объекта.
  • mask - массив слоёв, с которыми можно сталкиваться.
  • isTrigger - триггер без физического отталкивания.
  • touching.bottom и другие флаги - контакт по сторонам.

PhysicComponent

  • applyForce(x, y) - добавляет силу к текущему кадру.
  • velocity - текущая скорость по осям.
  • useGravity, gravityScale, drag.
  • isGrounded - приземлён ли объект после расчёта.
const playerBody = player.findComponent(PhysicComponent);
const playerCol = player.findComponent(ColliderComponent);

if (Engine.instance.input.isKeyPressed("Space") && playerBody.isGrounded) {
  playerBody.velocity.y = -650; // прыжок вверх
}

if (playerCol.isCollision) {
  console.log("Есть столкновение с:", playerCol.collision?.parent?.name);
}

8) Практические советы

  • Всегда вызывайте super.update(dt) в переопределённых update, чтобы не отключать логику компонентов.
  • Для детерминированного поведения учитывайте, что dt передаётся в секундах.
  • Если объект должен жить в конкретной сцене, создавайте его внутри Scene.init().
  • Для отладки коллизий храните ссылки на ColliderComponent и смотрите contacts/touching.
  • Используйте screenToWorld для кликов мышью в world-координатах.

9) Текущие ограничения и что учесть

  • Важно EventSystem пока заглушка и не используется в ядре.
  • Важно Ассеты в SpriteComponent берутся по ключу имени файла без расширения (например player.png -> ключ player).
  • Важно В текущей версии Engine.prepareData использует this.data; при работе с загрузкой изображений стоит проверить эту часть перед продакшеном.
  • Риск При ручном подключении скриптов соблюдайте порядок, иначе можно получить ошибки наследования классов.
